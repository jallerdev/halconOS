# Content Strategy SEO — Plan editorial para HalcónOS

> SEO no es magia — es producción consistente de contenido que responde a queries
> reales. Este es el plan: qué escribir, cuándo, y cómo medir si funciona.

## Filosofía

3 reglas:

1. **Escribe para ranking, no para tráfico ya conseguido**. Cada post debería tener una
   query target específica con volumen real.
2. **Un post largo > tres cortos**. 2000 palabras bien estructuradas rankea, 500 palabras
   genéricas no.
3. **Edita más de lo que escribes**. Un post viejo refrescado vale más que tres nuevos
   mediocres.

## Estructura por tipo de post

### Pillar pages (autoridad de tópico) — 1 cada 2 meses

- Longitud: 2500-3500 palabras.
- Estructura: cubre TODO sobre un tema (10+ H2 sections).
- Target: query genérica de mucho volumen (&quot;cómo conseguir clientes para una agencia&quot;).
- Linkea a 5-10 cluster posts.
- Update cada 6 meses.

### Cluster posts (long-tail) — 2 al mes

- Longitud: 1200-1800 palabras.
- Target: query específica con volumen medio (&quot;plantilla propuesta comercial agencia&quot;).
- Linkea al pillar correspondiente + 2-3 otros clusters.

### Comparativas / Listicles — 1 al mes

- Longitud: 1500-2500 palabras.
- Target: queries comerciales (&quot;HalcónOS vs HubSpot&quot;, &quot;mejores CRMs para agencias&quot;).
- Estos tienen el CTR más alto porque la gente está en mode-decisión.

## Cómo elegir queries (keyword research gratis)

### Tools sin pago

| Tool | Para qué |
|---|---|
| **Google Suggest** | Empieza a escribir en Google, ve las sugerencias automáticas |
| **People Also Ask** | Mira las preguntas debajo de resultados |
| **AnswerThePublic** | Genera preguntas comunes (5 búsquedas/día gratis) |
| **Ubersuggest** | Volumen estimado de búsquedas (free tier) |
| **Google Search Console** | Lo más valioso — queries reales que te llevan tráfico |

### Workflow para encontrar tu próxima query

1. Abre Ubersuggest gratis.
2. Pon un tema relacionado a HalcónOS (&quot;CRM agencia&quot;).
3. Filtra: volumen mensual 100-1000, dificultad 0-30.
4. Esta es tu lista de oportunidades.
5. Para CADA query, busca en Google y mira:
   - ¿Quién rankea? Si son sitios pequeños con DA <30, puedes competir.
   - ¿Qué formato responde mejor? (Lista, guía, comparativa).
   - ¿Qué falta en los actuales top 3? (Esa es tu oportunidad).

## Anatomía de un blog post que rankea

Cada post DEBE tener:

### Frontmatter (técnico)
- **Title** ≤60 chars, query target literal al inicio.
- **Description** 130-155 chars, vende el click.
- **Slug** kebab-case, sin acentos, máx 60 chars.
- **OG image** (auto-generada).
- **Keywords** array de 5-7 long-tails.
- **JSON-LD BlogPosting** (auto).

### Estructura del cuerpo

| Sección | Longitud | Por qué |
|---|---|---|
| H1 (título) | 1 línea | Inviolable: query target |
| Lead callout | 1 párrafo | Resumen + hook. Reduces bounce rate |
| H2: contexto / problema | 200 palabras | Conecta con el lector |
| H2: solución 1 (con H3s) | 400 palabras | Empieza meat |
| H2: solución 2 (con H3s) | 400 palabras | Variedad de respuestas |
| H2: ejemplos / tabla | 200 palabras | Visual + scannable |
| H2: errores comunes | 200 palabras | Diferenciador del top 3 |
| H2: próximos pasos | 150 palabras | Cierra con accionable |
| CTA HalcónOS | 1 párrafo | Soft sell, no agresivo |

**Total:** 1500-1800 palabras. Sweet spot para SEO + lectura.

### Internal links

Cada post debe linkear a 3-5 otros del mismo blog. Esto:
- Distribuye autoridad entre páginas.
- Aumenta time-on-site.
- Reduce bounce rate.

### Imágenes

Cada post debe tener:
- 1 imagen hero (OG image).
- 2-3 imágenes inline (screenshots, diagramas).
- TODAS con `alt` descriptivo (no &quot;screenshot.png&quot; — &quot;Captura del pipeline kanban con 5 leads en estado calificado&quot;).

## Métricas de éxito

### Mes 1
- Solo medir: que las páginas están **indexadas** en GSC.
- Tráfico esperado: 0-50 visitas/mes (Google indexa lento).

### Mes 3
- Empieza a ver tráfico orgánico medible.
- Target: 200-500 visitas/mes desde Google.
- Al menos 3 queries con posición top 30.

### Mes 6
- 1000-2000 visitas/mes.
- 10+ queries con posición top 10.
- 2-5 queries con posición top 3.

### Mes 12
- 5000-10000 visitas/mes (si todo va bien).
- Tráfico orgánico = lead generation predictible.

## Refresh: el secreto del SEO consistente

Cada 6 meses, revisa TODOS los posts y refresca los que perdieron posición:

1. GSC → Pages → mira posición promedio de cada post.
2. Si un post perdió ≥5 posiciones, refresh:
   - Agregar 200-400 palabras.
   - Mejorar el H1 si la query target cambió.
   - Actualizar dates / ejemplos / screenshots.
   - Cambiar `updatedAt` en el manifest.
3. Resubmit a GSC vía URL Inspection.

Un refresh bueno puede subir un post de posición 15 a posición 5 en 2-4 semanas.

## Resumen práctico

1. Escribe 2 posts al mes (1 cluster, 1 listicle/comparativa).
2. Cada 2 meses, 1 pillar de 2500+ palabras.
3. Cada 6 meses, refresh de los posts viejos.
4. Métricas: GSC weekly review.

Si haces esto consistente 12 meses, tienes **30 posts** que cubren tu mercado completo
y un canal de adquisición orgánico que NO depende de Ads.
